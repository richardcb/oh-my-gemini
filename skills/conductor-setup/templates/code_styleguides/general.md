# General Code Style Principles

These principles apply across all languages and frameworks.

## Readability

- Code should be easy to read and understand
- Avoid overly clever constructs
- Prefer explicit over implicit
- Use meaningful names that describe intent

## Consistency

- Follow existing patterns in the codebase
- Maintain consistent formatting and naming
- When in doubt, match what's already there

## Simplicity

- Prefer simple solutions over complex ones
- Break down complex problems into smaller parts
- Avoid premature optimization
- Don't over-engineer for hypothetical futures

## Maintainability

- Write code that is easy to modify
- Minimize dependencies and coupling
- Keep functions and files focused
- Make the code testable

## Documentation

- Document *why* something is done, not just *what*
- Keep documentation up-to-date
- Use comments sparingly - prefer self-documenting code
- Document edge cases and non-obvious decisions

## Error Handling

- Handle errors explicitly
- Provide meaningful error messages
- Fail fast and fail clearly
- Consider all failure modes

## Testing

- Write tests for new functionality
- Cover edge cases and error paths
- Keep tests readable and maintainable
- Tests are documentation too
